## ТЕОРЕТИЧНЕ ПИТАННЯ ВАРІАНТ - 2

NavMesh — це представлення ігрового простору у вигляді опуклих полігонів, по яких пересувається агент. На відміну від сітки клітинок, точніше описує поверхню і займає менше пам'яті.

1. **Побудова NavMesh** — геометрія рівня аналізується і з неї виділяються прохідні зони з урахуванням розміру агента, кута нахилу поверхні та перешкод. Результатом є сітка зв'язаних полігонів.
2. **Пошук шляху — A*** — полігони утворюють граф, по якому працює A*. Для кожного вузла рахується f = g + h, де g — вартість шляху від старту, h — евристика до цілі. Алгоритм завжди розкриває вузол з найменшим f — це гарантує оптимальний шлях.
3. **Згладжування (String Pulling)** — A* повертає послідовність полігонів, а не готовий шлях. Алгоритм String Pulling натягує лінію крізь канал полігонів і знаходить мінімальний плавний маршрут.
4. **Динамічні перешкоди** — для об'єктів що рухаються NavMesh оновлюється локально. Для уникнення зіткнень між агентами використовують RVO/ORCA — кожен агент самостійно обирає безпечну швидкість.
